Q-Translate is made of two parts, a client application and a server application, which communicate with each-other through an HTTP API. Both of these applications are built with Drupal7. For more flexibility, there is also a drupal module for the client, that can be installed independently on any Drupal application or distribution.

Installation is done inside a Docker container. For instructions about installing Docker see: https://docs.docker.com/installation/

Installing qtr_server
  1. Get the image of the system: 
    docker search qtranslate
docker pull qtranslate/qtr_server
  2. Create a container: 
    mkdir -p /qtr/downloads

### create a container
docker create --name=qtr --hostname=qtranslate.net \
    -v /qtr/downloads:/var/www/downloads \
    -p 80:80 -p 443:443 --restart=always \
    qtranslate/qtr_server

### start the container
docker start qtr
  3. Change the configuration: 
    ### enter the shell of the container
qtr-exec() { docker exec -it qtr env TERM=xterm script /dev/null -c "$@" -q; }
qtr-exec bash

### modify the configration settings
cp install/settings.sh qtr.conf
vim qtr.conf

### change the configuration
install/config.sh qtr.conf

### restart the container
exit
docker restart qtr
  4. Make more configuration changes: 
    ### change the domain
qtr-exec install/config/domain.sh

### change the supported languages
qtr-exec install/config/languages.sh

### change drupal admin password
qtr-exec install/config/drupalpass.sh

### change the gmail account
qtr-exec install/config/gmailsmtp.sh
  5. If this is a test installation or you are installing it on your local machine, add to /etc/hosts a line like this 127.0.0.1 qtranslate.net qtranslate.org dev.qtranslate.org with the domain that you used in configuration and the IP of the server. Now you can open in browser https://qtranslate.net https://qtranslate.org and access your application. You will have to confirm the security exception, since the SSL certificates of the web server are not real ones.
  6. Import all the translations: 
    qtr-exec /var/www/data/import.sh en
qtr-exec /var/www/data/import.sh fr
qtr-exec /var/www/data/import.sh de
qtr-exec /var/www/data/import.sh it

Installing qtr_client
  1. Get the image of the system: 
    docker search qtranslate
docker pull qtranslate/qtr_client
  2. Create a container: 
    ### create a container
docker create --name=qcl-en --hostname=en.qtranslate.net \
    -p 80:80 -p 443:443 --restart=always \
    qtranslate/qtr_client

### start the container
docker start qcl-en
  3. Change the configuration: 
    ### enter the shell of the container
docker-exec() { local container=$1; shift; docker exec -it $container env TERM=xterm script /dev/null -c "$@" -q; }
docker-exec qcl-en bash

### modify the configration settings
cp install/settings.sh qcl-en.conf
vim qcl-en.conf

### change the configuration
install/config.sh qcl-en.conf

### restart the container
exit
docker restart qcl-en
  4. If this is a test installation or you are installing it on your local machine, add to /etc/hosts a line like this 127.0.0.1 en.qtranslate.net with the domain that you used in configuration and the IP of the server. Now you can open in browser https://en.qtranslate.net and access the application. You will have to confirm the security exception, since the SSL certificates of the web server are not real ones.
  5. Make more configuration changes: 
    ### change the domain
docker-exec install/config/domain.sh

### change the translation language
docker-exec qcl-en install/config/translation_lng.sh

### change drupal admin password
docker-exec qcl-en install/config/drupalpass.sh

### change the gmail account
docker-exec qcl-en install/config/gmailsmtp.sh

### change the oauth2 login settings
docker-exec qcl-en install/config/oauth2_login.sh
